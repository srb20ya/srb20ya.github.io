LEVEL EDITING

All your basic sector creation, linedefs, etc. are all identical to Doom. There are a few extra things you have to watch out for, though:

* Avoid creating any IMPASSABLE linedefs that the player will come in contact with. Instead of using an impassable line, create a sector at least 60 units in width, and set its floor and ceiling heights to the same value.

* Don't use any walk, button, or shoot line triggers (WR, W1, SR, S1, DR, D1, etc.), as SRB2 does not utilize these. In fact, no line triggers are used whatsoever. Any special triggering is done by use of sector specials.

* I'll add more "Don't"s here when I think of them!

CUSTOM CHARACTERS

Yup, you can make your own characters to guide through SRB2! But you probably knew that. Anyway, this section covers absolutely everything you'd ever want to know about making your own character to use against your buddies.

Warning! This section is constantly changing and some things may be COMPLETELY different in the future! Keep up to date with this page to make sure your character works with the latest SRB2 version!

With that out of the way, let's begin!

Animations within a WAD file are named using four letters, for example, the default player is PLAY. The four other letters after it define the frame number and angle(s) it should be shown.

Letters are used to denote the frame number, and numeric digits (0-8) are used to define the angle.

For example:

PLAYA1 - First frame, shown at angle 1 (front view)

FLWRC0 - Third frame, show it for all angles (0 means show this sprite no matter what angle it is being looked at)

Note: The entries actually start at ASCII character 60 (A) and go forward. Wintex only supports capital letters, limiting you to a total of 32 frames (up to `). SRB2 goes beyond this (currently uses 33), and Wintex will ruin a wad if you save it without taking the right precautions. Don't blame us, it's these dirt-old utilities we've gotta put up with. =P

Before saving your file, rename any of the would-be lowercase entries at the end to something else that you can easily distinguish. For example, I'll rename:

KNUXa2a8

to

KNUKa2a8

(Note that Wintex will display the lowercase entires with a capital letter - which means you'll have a KNUXA2A8 at the beginning of your file and KNUXA2A8 near the end. The one near the end is the lowercase one.)

After saving with Wintex, it will convert it to KNUKA2A8. Next, crack open a Hex editor. If you cleaned your wad after saving, this will be easy to do. Scroll all the way down to the end of the file until you see the large list of entry names. You'll find, for example, KNUKA2A8 at the very end. Change it to KNUXa2a8.

It may sound hard, but it is really quite easy. Once you've done it for the first time, successive hex editing is painless.

Now that you've loaded all of your graphics into the entries and renamed the four-letter entries to something relevant to your character (i.e., you might name "Invader Zim" as IZIM), it's time to edit the S_SKIN data lump. This little tidbit provides information about your character, such as its name "Invader Zim", his speed, and abilities that he possesses. Let's not forget life icons and sounds, too!

S_SKIN is really just a text file, which looks like this:

name = Invader_Zim         <---- Spaces not allowed!

face = IZIMICON               <---- 32x32 life icon picture (must be 8 characters long)

facename = IZIMNAME      <---- Name graphic that appears above the life icon (must be 8 characters long)

ability = 1                        <---- Fly like Tails. Perhaps some kind of propeller can come out of his backpack

speed = 3                        <---- 40% of Sonic's speed. Zim can run, but he's no Sonic character

sprite = IZIM                    <---- Optional. Just reiterates which set of sprites to load to make sure the game doesn't get confused.

 

Here's the chart for all possible ability and speed numbers:

Ability
-------
0 = Sonic's Thok
1 = Tails' Fly
2 = Knuckles' Glide
3 = Nothing

If anything else is specified, it defaults to "Nothing".

Speed
-------
0 = Sonic's Speed (100%)
1 = Tails's Speed (60%)
2 = Knuckles's Speed (80%)
3 = 40% Speed
4 = 120% Speed
5 = 20% Speed

If anything else is specified, it defaults to "20% Speed".

 

To add custom sounds, you just list the original sound, and then the name of the custom sound in the WAD you want assigned to it. Here are the ones you can use:

DSSPIN - Sound when Sonic goes into a spin while moving.
DSPUTPUT - Tails flying
DSPUDPUD - Tails tired from flying
DSPLPAIN - Ring spill
DSPLDETH - "Ba-dum!" death sound
DSTHOK - Sound effect used for Sonic's Thok.
DSSPNDSH - Spindash rev
DSZOOM - Spindash release
DSRADIO - Little "ding" when you message a player
DSGASP - Grab a bubble
DSJUMP - Jump!

Note that your new sound MUST ALSO have DS as the first two letters.

So we would continue the text file like so:

 

dsspin = dszspin

dsputput = dszput

dspudpud = dszpud

dsplpain = dszpain

dspldeth = dszdeath

dsspndsh = dszdash

dszoom = dszzoom

dsradio = dsztalk

dsgasp = dszgasp

dsjump = dszjump

 

Notice that I didn't replace ALL the sounds. You don't have to. Only replace the ones you want.

The S_SKIN entry MUST be the FIRST one in your file and come before all of the sprites!

If you did everything right, it should work! (Zim is NOT a finished/downloadable character, sorry. =))
